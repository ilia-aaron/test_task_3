Привет, тебя приветствует команда разработки аналитических бизнес-приложений Практики бизнес и продвинутой аналитики [**Газпромнефть-ЦР**](https://ds.gazprom-neft.ru/)!

Это задание посложнее предыдущих — здесь нужно показать работу с большими данными, пагинацию и оптимизацию. Ближе к тому, с чем реально работаем на проекте.

## Что нужно сделать

Страница с таблицей проектов и панелью фильтров. В базе 10 000 записей — грузить всё разом на фронт нельзя, нужна серверная пагинация. Данные отдаёт json-server (уже настроен в проекте).

### Стек

- React 18 + TypeScript (strict mode)
- [AG Grid](https://www.ag-grid.com/react-data-grid/) v33 — таблица (используем на проекте). `ag-grid-enterprise` добавлен, можешь использовать Server-Side Row Model если хочешь, watermark в dev допустим
- [react-virtuoso](https://virtuoso.dev/) — виртуализация списка в фильтре менеджеров
- [Consta UI Kit](https://consta.design/libs/uikit) — для UI. **Не используй** `@consta/ag-grid-adapter` — он нестабилен, AG Grid стилизуй стандартными средствами
- json-server — API, уже настроен

### Запуск

```bash
npm install
npm run server    # API на localhost:3001
npm start         # приложение
```

### Данные

В базе два ресурса: 10 000 проектов (`/projects`) и 250 менеджеров (`/managers`).

Структура проекта:

- `id` (number), `projectName` (string), `department` (string — Разведка/Добыча/Переработка/Логистика/Сбыт)
- `status` (string — Активный/Завершён/Приостановлен/Планирование), `priority` (string — Высокий/Средний/Низкий)
- `budget` (number), `spent` (number), `progress` (number, 0-100)
- `startDate` (string, ISO), `managerId` (number), `manager` (string — ФИО)

Структура менеджера: `id`, `name` (Фамилия И.О.), `department`.

### API

**Проекты** `GET /projects`:
- Пагинация: `?_page=1&_limit=50` — в заголовке `X-Total-Count` общее кол-во
- Сортировка: `?_sort=budget&_order=desc`
- Фильтры: `?department=Добыча&status=Активный&manager=Иванов А.С.`

**Менеджеры** `GET /managers`:
- Поиск: `?name_like=Куз` (подстрока, найдет "Кузнецов" и т.д.)
- Пагинация: `?_page=1&_limit=20`

json-server также поддерживает `_ne` (не равно), `_like` (по регулярке) и другие операторы — полная документация [здесь](https://github.com/typicode/json-server/tree/v0).

### Готовое в проекте

В `src/ag-grid/` лежат:
- `theme.ts` — тема AG Grid под стилистику ГПН, подключи к гриду
- `locale-ru.ts` — русская локализация, подключи через `localeText`

## Требования

Главное — чтобы таблица работала с 10 000 записей и не тормозила. Конкретно:

1. **Серверная пагинация** — данные грузятся порциями, не всё сразу. Можно через AG Grid Server-Side Row Model, можно через свои контролы — на твой выбор

2. **Сортировка** по клику на заголовок — минимум бюджет и прогресс

3. **Фильтрация** по департаменту, статусу, приоритету (тут значений мало, можно захардкодить списки) + фильтр по менеджеру — тут сложнее, см. ниже

4. **Фильтр менеджеров** — их 250, не надо грузить все в дропдаун:
   - Поиск с debounce, подгрузка через API (`/managers?name_like=...`)
   - Виртуализация списка через react-virtuoso (при скролле подгружаются следующие страницы)
   - Чекбоксы — можно выбрать нескольких менеджеров
   - Мастер-чекбокс **"Выбрать все"** — должен выбрать **всех** менеджеров, не только те 20 что загрузились
   - Если в поиске что-то введено — "Выбрать все" выбирает всех подходящих
   - После выбора показывать количество в формате "N из 250" (точное число)

5. **Применение фильтров** по кнопке "Применить", не при каждом изменении. Кнопка "Сбросить" для очистки

6. **Consta UI Kit** — используй для кнопок, текста, дропдаунов

7. **TypeScript strict**, без `any` по возможности

### На что обратим внимание

В первую очередь — на фильтр менеджеров. Это не такая простая задача, как кажется на первый взгляд. Подумай, что произойдёт если пользователь:

- Нажмёт "Выбрать все" и применит фильтр — сколько параметров уйдёт в запрос?
- Выберет всех, потом снимет 2 — оптимально ли отправлять 248 параметров?
- Несколько раз поищет разное и каждый раз нажмёт "Выбрать все" — сохранится ли предыдущий выбор?
- Нажмёт "Выбрать все" и проскроллит список вниз — будут ли галочки у подгруженных элементов?

Нам важнее увидеть, как ты думаешь над этими кейсами, чем pixel-perfect вёрстку.
